# Docs Index

이 폴더는 팀원이 프로젝트를 빠르게 이해하기 위한 문서 모음입니다.

## 읽는 순서

1. `PROJECT_OVERVIEW.md`
- 프로젝트 목적, 데이터 규정, 현재 단계 요약

2. `PIPELINE.md`
- 어떤 스크립트를 어떤 순서로 실행하면 어떤 파일이 생기는지 설명

3. `EXPERIMENT_STATUS.md`
- 지금까지 어떤 실험을 했고, 현재 무엇이 가장 잘 나왔는지 요약

4. `COLLABORATION.md`
- 로컬, GitHub, EC2를 어떻게 나눠 쓰는지와 브랜치/커밋 규칙

## 이 문서들이 해결하려는 문제

이 프로젝트는 실제 실행 환경이 EC2이고, GitHub repo는 협업 기준점이며, 데이터는 Git에 올리지 않기 때문에 초반에 구조가 헷갈리기 쉽습니다.
이 문서들은 아래 질문에 답하도록 작성했습니다.

- 지금 코드의 공식 위치는 어디인가?
- EC2에 있는 파일과 GitHub repo의 파일은 어떤 관계인가?
- 어떤 스크립트가 최신 실험 코드인가?
- 지금 프로젝트가 어느 단계까지 진행됐는가?
- 내가 어디서 코드를 읽고, 수정하고, 커밋해야 하는가?
